BMS 之 ICON 进化史

过程

  1. ICON 1.0 用一个图标,引入一个库(library)
  2. 使用一个 ICON 自己打包字体文件，再引入
  3. ICON 2.0 用一个图标,写一个 Svg 模板
  4. ICON 3.0 用 Nebular 注册图标

好处

  • Nebular 负责将你传入的图标, 根据不同的 register font-icon pack, 来转换成对应的 SVG
  • Nebular 负责将你传入的图标进行统一的尺寸及样式管理 by its theme system
